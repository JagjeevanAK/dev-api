# Open Food Facts Documentation

![Open Food Facts Documentation](./assets/image.png)

Welcome to the comprehensive documentation for the Open Food Facts ecosystem. Our platform provides access to the world's largest collaborative food database and related services, powering applications that help people make better food choices.

## What is üçä Open Food Facts?

### A food products database

Open Food Facts is a database of food products with ingredients, allergens, nutrition facts and all the tidbits of information we can find on product labels.

### Made by everyone

Open Food Facts is a non-profit association of volunteers. 25,000+ contributors like you have added 1.7 million+ products from 150 countries using our Android, iPhone or Windows Phone app or their camera to scan barcodes and upload pictures of products and their labels.

### For everyone

Data about food is of public interest and has to be open. The complete database is published as open data and can be reused by anyone and for any use. All our APIs and services are free to use!

<Callout type="info">
  Open Food Facts is a collaborative, free and open database of food products
  from around the world. All data is available under open licenses, enabling
  innovation in food transparency.
</Callout>

Choose the documentation section that best fits your needs:

## Frequently Asked Questions

<Accordions type="single">
  <Accordion title="Do I need authentication to use the APIs?" id="auth-question">
    Most of our APIs are free to use without authentication for read operations. Authentication is only required for write operations (adding products, editing data, etc.). Check the specific API documentation for authentication requirements.
  </Accordion>
  
  <Accordion title="What data formats are supported?" id="data-formats">
    Our APIs support multiple data formats including JSON, JSONL, CSV, and XML. The Product Opener API returns JSON by default, but you can request specific formats using the appropriate endpoints or parameters.
  </Accordion>
  
  <Accordion title="Is there a rate limit for API calls?" id="rate-limits">
    We don't enforce strict rate limits for reasonable usage, but we ask developers to be respectful of our resources. For high-volume applications, please contact us to discuss your needs and ensure optimal performance.
  </Accordion>
  
  <Accordion title="How can I contribute to Open Food Facts?" id="how-to-contribute">
    There are many ways to contribute:
    - **Add Products**: Use our mobile apps to scan and add products
    - **Code Contributions**: Check our GitHub repositories for open issues
    - **Translation**: Help translate the interface at translate.openfoodfacts.org
    - **Documentation**: Improve our documentation and guides
    - **Data Quality**: Help moderate and improve existing product data
  </Accordion>
  
  <Accordion title="What licenses apply to the data and code?" id="licenses">
    - **Data**: Open Database License (ODbL) - you can use, modify, and redistribute the data
    - **Code**: Most projects use AGPL license, some components use other open-source licenses
    - **Images**: Various licenses depending on the source, check individual image metadata
  </Accordion>
  
  <Accordion title="How do I report bugs or request features?" id="bugs-features">
    - **GitHub Issues**: Report bugs and request features on the specific project repository
    - **Community Forum**: Discuss ideas and get help from the community
    - **Developer Slack**: Join our Slack for real-time discussion with developers
  </Accordion>
</Accordions>

## Technical Details

<Accordions type="single">
  <Accordion title="System Architecture Overview" id="architecture">
    Open Food Facts uses a microservices architecture with the following main components:
    - **Product Opener**: Core Perl-based backend handling the main database and web interface
    - **Robotoff**: Python-based AI service for automated data extraction
    - **Search-a-licious**: Elasticsearch-based search infrastructure
    - **Knowledge Panels**: FastAPI service for structured product information
    - **Open Prices**: Price tracking database and API
    - **Folksonomy Engine**: Collaborative tagging system
  </Accordion>
  
  <Accordion title="Database Schema and Data Model" id="database-schema">
    Our database uses a flexible document-based model with MongoDB as the primary storage:
    - **Products**: Core product information including ingredients, nutrition facts, images
    - **Users**: Contributor accounts and authentication data
    - **Revisions**: Complete history of all product changes
    - **Images**: Product photos with metadata and processing information
    - **Taxonomies**: Hierarchical classification systems for ingredients, categories, labels
  </Accordion>
  
  <Accordion title="Performance and Scalability" id="performance">
    Open Food Facts serves millions of users worldwide:
    - **CDN**: Global content delivery network for images and static assets
    - **Caching**: Redis-based caching for frequently accessed data
    - **Load Balancing**: Multiple server instances with traffic distribution
    - **Database Optimization**: Indexed queries and efficient data structures
    - **API Optimization**: Compressed responses and efficient serialization
  </Accordion>
</Accordions>
